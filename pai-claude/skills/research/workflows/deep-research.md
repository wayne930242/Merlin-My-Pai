# 深度研究 Workflow

## 流程

### Phase 1: 問題定義 (5-10 min)

1. 明確研究問題
2. 確定成功標準
3. 設定範圍限制
4. 列出已知資訊

### Phase 2: 資料收集 (30-60 min)

1. **官方來源**
   - 官方文檔
   - GitHub repo
   - 官方部落格

2. **社群來源**
   - Stack Overflow
   - Reddit
   - Hacker News

3. **實作經驗**
   - 技術部落格
   - Case studies
   - 開源專案

### Phase 3: 分析整理 (20-30 min)

1. 建立比較矩陣
2. 識別關鍵差異
3. 評估適用性
4. 記錄不確定性

### Phase 4: 結論撰寫 (10-15 min)

1. 總結發現
2. 提出建議
3. 列出後續行動
4. 記錄參考資料
