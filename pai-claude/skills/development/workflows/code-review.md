# Code Review Workflow

## 檢查清單

### 功能性
- [ ] 程式碼是否正確實作需求？
- [ ] 邊界情況是否處理？
- [ ] 錯誤處理是否完整？

### 可讀性
- [ ] 命名是否清晰？
- [ ] 邏輯是否容易理解？
- [ ] 註解是否必要且有幫助？

### 維護性
- [ ] 是否遵循 DRY 原則？
- [ ] 函數是否單一職責？
- [ ] 是否容易測試？

### 效能
- [ ] 是否有不必要的計算？
- [ ] 資料結構是否合適？
- [ ] 是否有 N+1 問題？

### 安全性
- [ ] 輸入是否驗證？
- [ ] 是否有注入風險？
- [ ] 敏感資料是否保護？

## Review 回饋格式

```markdown
## 總結
[整體評價]

## 必須修改
- [ ] [問題描述] (檔案:行數)

## 建議改進
- [ ] [建議描述]

## 優點
- [值得學習的地方]
```
